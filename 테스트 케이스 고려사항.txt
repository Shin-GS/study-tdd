테스트 케이스 세분화

1)해피 케이스
 - 아무것도 잘못되지 않는 사용자 시나리오
 - 의도에서 벗어난 행동이 없는 케이스

2)예외 케이스
 - 작업자가 의도한 범위에서 벗어난 행동을 하는 케이스

3)경계값 테스트 고려 필요
 - 범위(이상, 이하, 초과, 미만), 구간, 날짜 등

4)테스트하기 어려운 영역 확인
 - 관측할 때마다 다른 값에 의존하는 코드
   현재날짜/시간, 랜덤값, 전역변수/함수, 사용자 입력 등
 - 외부 세계에 영향을 주는 코드
   표준출력, 메시지발송, 데이터베이스에 기록하기 등

5)테스트하기 쉬운 영역 확인
 - 같은 입력에는 항상 같은 결과를 반환하는 코드
 - 외부 세상과 단절된 형태의 코드
 - 순수 함수

*통합 테스트
 - 여러 모듈이 협력하는 기능을 통합적으로 검증하는 테스트
 - 일반적으로 작은 범위의 단위 테스트만으로는 기능 전체의 신뢰성을 보장할 수 없다.
 - 풍부한 단위 테스트 & 큰 기능 단위를 검증하는 통합 테스트
